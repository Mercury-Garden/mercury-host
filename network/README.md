# network/

Tracked copies of this host's network configuration, plus reference
documentation about the Oracle Cloud VCN it lives in.

## What's tracked

- **`hostname`** — exact copy of `/etc/hostname` (just `mercury`).
  No trailing newline; single-token file per `/etc/hostname` spec.
- **`hosts`** — exact copy of `/etc/hosts`. Includes the IPv6
  multicast prefix entries and the `127.0.1.1 mercury` line that
  some Debian/Ubuntu installers add.
- **`vcn-topology.md`** — hand-written reference for the Oracle Cloud
  VCN, subnet, route table, and ingress path. Not auto-generated.
  Update this when the VCN layout changes (e.g., new subnet added,
  security list rule added, public IP changed).

## What's NOT tracked (and why)

| Thing | Why not |
|---|---|
| `/etc/resolv.conf` | Generated by `systemd-resolved` and DHCP; moves around |
| OCI VCN/subnet/route table IDs | Visible only in OCI Console; not a file on this host |
| Public IP for `mercury.garden` | OCI-assigned; can change if instance is rebuilt |
| TLS certs / private keys | See `secrets/inventory.yaml`; never commit |

## Capture / restore

- **`scripts/capture.sh`** regenerates `hostname` and `hosts` from `hostname`
  and `/etc/hosts` respectively.
- **`scripts/restore.sh`** would copy these back to `/etc/hostname` and
  `/etc/hosts`. Requires `sudo`. Not yet wired up.
- **`scripts/audit.sh`** compares `$(hostname)` against `network/hostname`
  and `diff -q /etc/hosts network/hosts` — drift reported as a `note`
  (informational, not a hard error, since capture.sh can refresh on next run).

## Reference docs in this directory

- `vcn-topology.md` — start here for "what subnet is this host on?"
- `../host.yaml` — single-line hostname/IP/OS summary (machine-readable)
- `../inventory.yaml` — full inventory including all `mercury.garden`
  subdomain bindings and the nginx service that fronts them
- `../nginx/sites-available/*.conf` — per-subdomain nginx configs
  (TLS cert paths, proxy_pass targets, auth_request directives)
